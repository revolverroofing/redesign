# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

> The `@AGENTS.md` import above pulls in a critical warning: this project pins **Next.js 16.2.4**, which has breaking changes vs. older Next.js conventions in training data. Before writing routing, data-fetching, caching, or `<Image>` code, check `node_modules/next/dist/docs/` for the version-accurate guide rather than relying on memory.

## Stack

- **Next.js 16.2.4** (App Router, `src/app`) on **React 19.2**
- **TypeScript 5** (strict, `noEmit`, `bundler` module resolution)
- **Tailwind CSS v4** — PostCSS-only, *no* `tailwind.config.ts`. Theme tokens are declared inline in CSS via `@theme inline {}` (see `src/app/globals.css`). Adding a v3-style JS config will not be picked up.
- **ESLint 9 flat config** (`eslint.config.mjs`) extending `eslint-config-next` (`core-web-vitals` + `typescript`)
- **pnpm** is the package manager. `pnpm-workspace.yaml` exists only to allowlist native build deps (`sharp`, `unrs-resolver`); this is a single-package repo, not a workspace.

## Commands

```bash
pnpm install            # install deps
pnpm dev                # next dev (http://localhost:3000)
pnpm build              # next build (uses Turbopack)
pnpm start              # serve the production build
pnpm lint               # eslint (flat config, no extra args needed)
pnpm exec tsc --noEmit  # standalone typecheck (no script alias)
```

There is **no test runner configured**. If you add tests, also wire a `test` script and document the single-test invocation here.

## Architecture

The app is a clean App Router scaffold with one route:

- `src/app/layout.tsx` — root layout. Loads Geist + Geist Mono via `next/font/google`, exposes them as the `--font-geist-sans` / `--font-geist-mono` CSS variables on `<html>`, and sets the page to a flex column so children can fill the viewport.
- `src/app/page.tsx` — the only route (`/`). Currently the create-next-app placeholder; replace with real homepage content.
- `src/app/globals.css` — single global stylesheet. Imports Tailwind (`@import "tailwindcss"`) and defines the design tokens (`--background`, `--foreground`, font families) inside `@theme inline`. Light/dark are driven by `prefers-color-scheme`, not a class toggle — switch to a `data-theme`/`class` strategy if a manual theme switcher is needed.
- `public/` — static assets served at `/`.
- `@/*` import alias resolves to `src/*` (see `tsconfig.json`).

When adding routes, prefer colocated server components in `src/app/<route>/page.tsx`; only mark files `"use client"` when they need browser-only APIs or interactivity.

## Conventions

- **Branch flow**: AI-assisted work lives on `claude/<topic>-<suffix>` branches and is pushed there, never directly to `main`.
- **Tailwind v4 idioms**: extend the design system by editing the `@theme inline` block in `globals.css`, not by creating a JS config. Reference tokens as `bg-background`, `text-foreground`, `font-sans`, etc.
- **Fonts**: use the `--font-geist-*` variables that `layout.tsx` already exposes. Note that `globals.css` currently sets `body { font-family: Arial... }`, which overrides Geist on `<body>`; remove that rule (or switch it to `var(--font-sans)`) when you want Geist applied site-wide.
- **next-env.d.ts** is generated by `next dev`/`build` and is gitignored — do not hand-edit or commit it.
